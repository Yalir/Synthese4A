00127         /*
00128          * This boolean was added to eliminate a heinous bug where we would
00129          * get into a loop with the server and move a buddy back and forth
00130          * from one group to another.
00131          *
00132          * The sequence goes something like this:
00133          * 1. Our resource and another resource both approve an authorization
00134          *    request at the exact same time.  We put the buddy in group A and
00135          *    the other resource put the buddy in group B.
00136          * 2. The server receives the roster add for group B and sends us a
00137          *    roster push.
00138          * 3. We receive this roster push and modify our local blist.  This
00139          *    triggers us to send a roster add for group B.
00140          * 4. The server recieves our earlier roster add for group A and sends
00141          *    us a roster push.
00142          * 5. We receive this roster push and modify our local blist.  This
00143          *    triggers us to send a roster add for group A.
00144          * 6. The server receives our earlier roster add for group B and sends
00145          *    us a roster push.
00146          * (repeat steps 3 through 6 ad infinitum)
00147          *
00148          * This boolean is used to short-circuit the sending of a roster add
00149          * when we receive a roster push.
00150          *
00151          * See these bug reports:
00152          * http://trac.adiumx.com/ticket/8834
00153          * http://developer.pidgin.im/ticket/5484
00154          * http://developer.pidgin.im/ticket/6188
00155          */
00156         gboolean currently_parsing_roster_push;
